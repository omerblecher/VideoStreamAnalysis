2. `ShmFrameMessage` ŌĆö a dataclass (or namedtuple) carrying everything the Detector needs to find the frame:

  <name>Task 2: Streamer process</name>

  <action>
Create `streamer.py` implementing the Streamer process function `run_streamer(video_path: str, to_detector: Queue)`.

Algorithm:
1. Open `cv2.VideoCapture(video_path)`. If `cap.isOpened()` is False, put `EOS_SENTINEL` on `to_detector` and return immediately (downstream won't hang).
2. Read FPS: `fps = cap.get(cv2.CAP_PROP_FPS)`. If fps <= 0 fallback to 25.0.
3. Loop: `ret, frame = cap.read()`. When `ret` is False (video exhausted), break.
4. For each frame:
   a. Allocate a new `SharedMemory` block: `size = frame_shm_size(frame.shape, frame.dtype)`, `shm = SharedMemory(create=True, size=size)`.
   b. Write frame bytes: `np.ndarray(frame.shape, dtype=frame.dtype, buffer=shm.buf)[:] = frame`.
   c. Close the local handle (do NOT unlink yet ŌĆö Detector still needs it): `shm.close()`.
   d. Build `ShmFrameMessage(shm_name=shm.name, frame_shape=frame.shape, frame_dtype=str(frame.dtype), frame_index=counter, fps=fps)`.
   e. Put message on `to_detector`.
   f. Increment counter.
5. After loop: put `EOS_SENTINEL` on `to_detector`. Release cap.

SharedMemory lifecycle note: The block is created here (Streamer), forwarded through Detector to Viewer, and must be unlinked by the Viewer after it finishes displaying that frame. Streamer closes its handle but does NOT unlink. Add a comment explaining this.

Imports: `cv2`, `numpy as np`, `multiprocessing.shared_memory.SharedMemory`, `multiprocessing.Queue`, `ipc` module items.

No `if __name__ == "__main__"` block needed ŌĆö this is imported by main.py.
