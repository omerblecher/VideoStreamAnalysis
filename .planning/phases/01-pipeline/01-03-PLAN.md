Create `viewer.py` implementing `run_viewer(from_detector: Queue, video_path: str)`.

`video_path` is passed in so the Viewer can display the filename in the window title.

Loop body:
1. `msg = from_detector.get()` — blocks until message available.
2. If `msg is EOS_SENTINEL`: call `cv2.destroyAllWindows()` and return.
3. Attach to SharedMemory: `shm = SharedMemory(name=msg.shm_name, create=False)`.
4. Reconstruct frame: `frame = np.ndarray(msg.frame_shape, dtype=msg.frame_dtype, buffer=shm.buf).copy()` — copy immediately so we can close shm.
5. Close and unlink: `shm.close(); shm.unlink()` — Viewer is the last consumer, so it frees the block.
6. Draw bounding rectangles per user decisions:
   - For each contour `c` in `msg.contours`: compute `x, y, w, h = cv2.boundingRect(c)`.
   - Draw: `cv2.rectangle(frame, (x, y), (x+w, y+h), (0, 255, 0), 2)` — green, 2px.
   - Label: `cv2.putText(frame, "motion", (x, y - 5), cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 1)` — green label above box.
7. Timestamp overlay (per user decisions — white text on dark background):
   - Compute elapsed seconds from frame_index and fps: `elapsed = int(msg.frame_index / msg.fps)`. Format: `ts = f"{elapsed // 3600:02d}:{(elapsed % 3600) // 60:02d}:{elapsed % 60:02d}"`.
   - Draw dark background rectangle behind text: measure text size with `cv2.getTextSize`, draw filled black rect with slight padding.
   - Draw white text: `cv2.putText(frame, ts, (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)`.
8. Window title: set once before loop using a module-level variable or function arg. Use: `title = f"VideoStreamAnalysis — {os.path.basename(video_path)} | {msg.fps:.0f}fps"`. Use `cv2.namedWindow(title, cv2.WINDOW_NORMAL)` before loop, `cv2.imshow(title, frame)`.
9. Frame rate control (per user decisions — never drop frames):
   - `delay = max(1, int(1000 / msg.fps))` — milliseconds per frame.
   - `cv2.waitKey(delay)`.

Window initialization: Before the loop, set `title` based on video_path. Call `cv2.namedWindow(title, cv2.WINDOW_NORMAL)` once. Use the same `title` string in every `cv2.imshow`.

Imports: `cv2`, `numpy as np`, `os`, `multiprocessing.shared_memory.SharedMemory`, `multiprocessing.Queue`, `from ipc import DetectorMessage, EOS_SENTINEL`.

Create `main.py` as the single entry point for the pipeline.

Structure:

```python
import sys
import os
import multiprocessing

from streamer import run_streamer
from detector import run_detector
from viewer import run_viewer

def main():
    if len(sys.argv) != 2:
        print("Usage: python main.py <video_path>")
        sys.exit(1)

    video_path = sys.argv[1]

    if not os.path.isfile(video_path):
        print(f"Error: video file not found: {video_path}")
        sys.exit(1)

    to_detector = multiprocessing.Queue()
    to_viewer = multiprocessing.Queue()

    streamer_proc = multiprocessing.Process(target=run_streamer, args=(video_path, to_detector), name="Streamer")
    detector_proc = multiprocessing.Process(target=run_detector, args=(to_detector, to_viewer), name="Detector")
    viewer_proc   = multiprocessing.Process(target=run_viewer,   args=(to_viewer, video_path),  name="Viewer")

    for p in [streamer_proc, detector_proc, viewer_proc]:
        p.start()

    for p in [streamer_proc, detector_proc, viewer_proc]:
        p.join()

if __name__ == "__main__":
    main()
```

Notes:
- Queue `maxsize` is NOT set (unbounded) — this allows Streamer to run ahead of Detector without blocking. Natural backpressure from Python's GIL and memory is acceptable for this demonstration project.
- `multiprocessing.Process` `name` args are for debugging readability only.
- `p.join()` waits for natural termination (EOS propagation handles shutdown, which is Phase 3 scope — in Phase 1, processes terminate when Viewer returns after EOS).
- The `if __name__ == "__main__":` guard is required for multiprocessing on Windows (spawn start method).

Do NOT add `multiprocessing.set_start_method("spawn")` — it's the default on Windows and calling it again raises an error if already set.
