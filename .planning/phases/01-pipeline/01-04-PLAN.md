3. Commit with a message explaining the SharedMemory design decision (required by GIT-03):

```
git -C /c/code/VideoStreamAnalysis commit -m "feat(phase-a): three-process pipeline with SharedMemory IPC

Design decision: Frame data transferred via multiprocessing.SharedMemory
instead of Queue.put(frame) to avoid copying ~900KB BGR arrays per frame
between processes. Each frame is written once into a SharedMemory block by
Streamer; the block name (string) is passed through Queue messages so
Detector and Viewer can attach to the same memory without copying. Viewer
is the final consumer and calls shm.unlink() to free each block.

Queue carries only lightweight metadata: ShmFrameMessage (shm_name,
shape, dtype, frame_index, fps) and DetectorMessage (same fields +
contours list). This separates bulk data transfer (SharedMemory) from
coordination signals (Queue), giving natural backpressure without the
overhead of serializing large NumPy arrays.

Implements: PIPE-01 through PIPE-07, IPC-01, IPC-02, IPC-03"
```
