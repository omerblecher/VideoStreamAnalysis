  <name>Task 2: Update main.py with stop_event, join timeout, force-terminate, and try/finally cleanup</name>
  <files>main.py</files>
  <action>
    Rewrite main() in main.py to implement the shutdown protocol from 03-CONTEXT.md decisions:

    1. Create the stop_event:
       stop_event = multiprocessing.Event()

    2. Pass stop_event to all three Process targets:
       - streamer_proc: args=(video_path, to_detector, release_queue, stop_event)
       - detector_proc: args=(to_detector, to_viewer, stop_event)
       - viewer_proc:   args=(to_viewer, video_path, release_queue, stop_event)

    3. Define JOIN_TIMEOUT = 5 (module-level constant, seconds)

    4. Replace the bare `for p in [...]: p.join()` with a try/finally block:

       SHUTDOWN_TIMEOUT = 5  # seconds; constant at module level

       try:
           for p in [streamer_proc, detector_proc, viewer_proc]:
               p.start()

           for p in [streamer_proc, detector_proc, viewer_proc]:
               p.join(timeout=SHUTDOWN_TIMEOUT)
               if p.is_alive():
                   print(f"[main] WARNING: {p.name} did not exit within {SHUTDOWN_TIMEOUT}s — terminating")
                   p.terminate()
                   p.join()
       finally:
           # Best-effort cleanup for any SharedMemory blocks left in release_queue
           # (handles the abnormal-exit case where Streamer never drained its open_handles).
           # On normal exit this queue is empty; on abnormal exit we log but don't raise.
           drained = 0
           while not release_queue.empty():
               try:
                   release_queue.get_nowait()
                   drained += 1
               except Exception:
                   break
           if drained:
               print(f"[main] WARNING: drained {drained} unreleased shm signal(s) from release_queue")

    Notes:
    - The per-frame SharedMemory lifecycle (Viewer closes+unlinks each block, Streamer closes its handle after release signal) is already self-cleaning on normal exit. The finally block handles the edge case where that loop never completed.
    - Import multiprocessing.Event is available from the existing `import multiprocessing` line — no new imports needed.
    - SHUTDOWN_TIMEOUT should be a module-level constant (not inside main()), placed after the imports block.
    - Keep the existing sys/os/multiprocessing imports and process definitions style.
    - Log warning lines use print with [main] prefix for clarity.
  </action>
